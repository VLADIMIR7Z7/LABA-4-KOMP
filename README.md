## Цель работы

## Постановка задачи

Необходимо разработать модуль поиска подстрок с использованием регулярных выражений и встроить его в текстовый редактор.

Программа должна:
- выполнять поиск подстрок по выбранному шаблону;
- отображать результаты поиска в таблице;
- выводить для каждого совпадения найденную подстроку, номер строки, номер символа и длину;
- отображать количество найденных совпадений;

1. Построить регулярное выражение для логина

2. Построить регулярное выражение для поиска переменной в стиле camelCase  
   (начинается со строчной буквы, затем буквы и цифры)

   Формат даты: `YYYY-MM-DD`

# Решение задач

## Задача 1. Логин

### Описание задачи

Необходимо построить регулярное выражение для поиска логина. Логин должен начинаться с латинской буквы, содержать латинские буквы, цифры, точку и дефис, а также не начинаться с цифры.

### Регулярное выражение

```regex
^[A-Za-z][A-Za-z0-9.-]*$
```

### Пояснение каждого обозначения

- `^` — начало строки;
- `[A-Za-z]` — первый символ должен быть латинской буквой;
- `[A-Za-z0-9.-]*` — далее может идти любое количество латинских букв, цифр, точек или дефисов;
- `*` — повторение предыдущего набора символов ноль или более раз;
- `$` — конец строки.

### Примеры строк, которые должны находиться

```text
my-login
alpha.beta
test-user
user1
test-2
camelCase
simpleVar2
x
badCamelCase
CamelCase
```

### Примеры строк, которые не должны находиться

```text
123login
-login
bad_login
```

### Тестовые примеры

Для проверки использовались строки:

```text
my-login
alpha.beta
test-user
user1
test-2

bad_login
123login
-login
```

Ожидаемый результат поиска:
- должны быть найдены: `my-login`, `alpha.beta`, `test-user`, `user1`, `test-2`;
- также будут найдены строки вида `camelCase`, `simpleVar2`, `x`, `badCamelCase`, `CamelCase`, так как они тоже удовлетворяют условию логина;
- не должны быть найдены: `bad_login`, `123login`, `-login`.

## Задача 3. Дата YYYY-MM-DD с учетом високосных годов

### Описание задачи

Необходимо построить регулярное выражение для проверки корректности даты в формате `YYYY-MM-DD` с учетом:
- количества дней в каждом месяце;
- особенностей февраля;
- високосных годов.

### Регулярное выражение

```regex
^(?:(?:(?:[0-9]{4}-(?:(?:01|03|05|07|08|10|12)-(?:0[1-9]|[12][0-9]|3[01])|(?:04|06|09|11)-(?:0[1-9]|[12][0-9]|30)|02-(?:0[1-9]|1[0-9]|2[0-8]))))|(?:(?:[0-9]{2}(?:0[48]|[2468][048]|[13579][26])|(?:0[48]|[2468][048]|[13579][26])00)-02-29))$
```

### Пояснение каждого обозначения

- `^` — начало строки;
- `[0-9]{4}` — год из четырех цифр;
- `(01|03|05|07|08|10|12)` — месяцы, в которых 31 день;
- `(04|06|09|11)` — месяцы, в которых 30 дней;
- `02` — февраль;
- `(0[1-9]|[12][0-9]|3[01])` — допустимые дни от 01 до 31;
- `(0[1-9]|[12][0-9]|30)` — допустимые дни от 01 до 30;
- `(0[1-9]|1[0-9]|2[0-8])` — допустимые дни февраля от 01 до 28;
- отдельная часть выражения описывает 29 февраля для високосных годов;
- `$` — конец строки.

### Примеры строк, которые должны находиться

```text
2024-02-29
2023-02-28
2025-12-31
```

### Примеры строк, которые не должны находиться

```text
2023-02-29
1900-02-29
2024-13-01
2024-00-10
```

### Тестовые примеры

Для проверки использовались строки:

```text
2024-02-29
2023-02-28
2025-12-31

2023-02-29
1900-02-29
2024-13-01
2024-00-10
```

Ожидаемый результат поиска:
- должны быть найдены: `2024-02-29`, `2023-02-28`, `2025-12-31`;
- не должны быть найдены: `2023-02-29`, `1900-02-29`, `2024-13-01`, `2024-00-10`.

### Доп задание БЛОК 2
В рамках дополнительного задания реализован алгоритм поиска подстрок через граф конечного автомата для задачи 2 (поиск переменной в стиле camelCase).

### Описание реализации

Для проверки строки на соответствие формату camelCase вместо регулярного выражения был реализован конечный автомат.

Автомат содержит следующие состояния:

- `Start` — начальное состояние;
- `FirstLowercase` — считана первая строчная латинская буква;
- `LettersOrDigits` — считываются допустимые символы (буквы и цифры);
- `Error` — состояние ошибки при обнаружении недопустимого символа.

### Логика работы автомата

1. В начальном состоянии ожидается первый символ строки:
   - если это строчная латинская буква (`a–z`), выполняется переход в состояние `FirstLowercase`;
   - иначе строка считается некорректной.

2. В состоянии `FirstLowercase` и `LettersOrDigits`:
   - допускаются латинские буквы (`a–z`, `A–Z`) и цифры (`0–9`);
   - при встрече любого другого символа выполняется переход в состояние `Error`.

3. Если в процессе обработки достигнуто состояние `Error`, строка отклоняется.

4. Строка считается корректной, если автомат завершил работу в состоянии:
   - `FirstLowercase` или
   - `LettersOrDigits`.
